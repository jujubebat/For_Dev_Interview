# Java

**Checked Exception**

- Exception을 상속하는 예외다.
- 컴파일 시점에 예외에 대한 처리(try/catch or throws)를 하지 않을 경우 컴파일 에러가 발생한다.
- 호출하는 메소드가 Exception을 활용해 무엇인가 의미 있는 작업을 할 수 있다면 Checked Exception을 사용하자

**Unchecked Exception**

- RuntimeException을 상속하는 예외다.
- 예외처리를 필수적으로 하지 않아도 된다.
- 호출하는 메소드가 Exception을 처리할 수 없는 상황이라면  Unchecked Exception을 사용하자
- 명확하지 않다면 Unchecked Exception을 사용하자

**final 키워드**

- 변수 앞에 final 키워드를 붙여주면 상수가 된다.
- 상수 변수에는 값을 딱 한 번만 할당 할 수 있고, 할당 된 값은 변경이 불가능하다.
- final을 통해 값을 고정 시키면 프로그램에 안정성을 높일 수 있다.
- 클래스 앞에 final 키워드를 붙여주면 해당 클래스는 상속이 불가능하다.
- 메소드 앞에 final 키워드를 붙여주면 오버라이딩이 허용되지 않는다.

**클래스, 객체(인스턴스)**

- 클래스는 객체를 만드는 틀이다. 
- 클래스는 객체를 관리하는 관리자다.
- 클래스는 객체를 생성하며 일반적으로 클래스가 객체를 "인스턴스화 한다."라고 표현한다.

**가변 객체 (Mutable)**

- 상태를 변경할 수 있는 객체다.

**불변 객체(Immutable)**

- 상태를 변경할 수 없는 객체다.
- 상태를 변경한다면, 원본은 바뀌지 않고 새로운 객체가 생성된다.
- 불변 객체는 변하지 않기 때문에 여러 쓰레드에서 사용되도 문제가 되지 않는다. 즉, 스레드 안전하다.

**자바의 형변환 규칙**

- 자동 형 변환 규칙
  - 자료형이 다른 두 변수를 연산할 때 자료형의 크기가 큰 방향으로 자동으로 형 변환이 이루어진다.
  - 예를들어 long형 변수와 int형 변수를 더하면 int형 변수가 long으로 변환된다.
- 명시적 형 변환 규칙
  - 소괄호 기호를 사용해서 자료형을 변환할 수 있다. 

**equals**

- == 연산자는 객체의 참조 값을 비교한다.
- 두 객체를 내용을 기준으로 비교하려면 equals 메서드를 재정의 해야한다.
- 두 객체가 물리적으로 같은지가 아니라 논리적 동치성을 확인해야 할 때 재정의 해줘야한다.

**Wrapper class** 

- 기본 자료형의 값을 감싸는 클래스다.
- 값을 인스턴스에 갑싸는 행위를 Boxing이라고한다.
- 반대로 저장된 값을 꺼내는 행위를 UnBoxing이라고 한다.
- 자바 5부터는 박싱과 언박싱이 필요한 상황에서 이를 자동으로 처리한다.

- 박싱된 기본 타입은 값에 더해 식별성(identity)이란 속성을 갖는다.
- 박싱된 기본 타입은 유효하지 않은 값, 즉 null을 가질 수 있다.
- 기본 타입이 박싱된 기본 타입보다 시간과 메모리 사용면에서 더 효율적이다.

**static 키워드**

- static 키워드를 변수, 메서드 앞에 붙여서 클래스 변수, 클래스 메서드를 선언할 수 있다. 
- 클래스 변수와 클래스 메서드는 메모리 공간에 딱 하나만 존재하며 클래스의 모든 인스턴스가 공유한다.
- 클래스 변수와 클래스 메서드는 인스턴스 생성 여부와 상관없이 JVM이 클래스를 로딩하는 순간 메모리 공간(메소드 영역)에 할당된다.

**접근 제어자**

- 접근 제어자는 클래스와 인스턴스 멤버를 대상으로 접근 허용 범위를 결정한다.
- public은 모든 범위
- default는 동일 패키지
- protected는 상속받은 클래스
- private은 클래스 내부
